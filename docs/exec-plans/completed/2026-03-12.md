## v136 — Gitignore E2E artifacts, extract base.py magic numbers, _truncate_summary validation

**Status:** Completed
**Created:** 2026-03-12

## Goal

Three self-contained improvements:

1. **Gitignore E2E test artifacts** — E2E tests create UUID-named directories at the repo root (e.g. `fe5fbd7c-b8bd-4444-a434-eced48ecf4e0/git/owner_repo/HELPING_HANDS_E2E.md`) that persist after test runs and can be accidentally committed. Add gitignore pattern for UUID directories. Resolves the TODO.md item.
2. **Extract magic numbers in base.py** — `200` (file list display limit, error preview length) and `4000` (precommit output limit) are hardcoded in `Hand` base class; `2000` (apply changes summary limit) is hardcoded in `_CLIBasedIterativeHand`. Extract to module-level constants for maintainability, consistent with v135's approach for mcp_server.py and celery_app.py.
3. **`_truncate_summary` limit validation** — accepts `limit: int` without checking `limit > 0`. A zero or negative limit would produce unexpected behavior. Add `ValueError` guard.
