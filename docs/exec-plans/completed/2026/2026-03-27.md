## v313 — Multiplayer Performance & Backend Awareness Fix

**Backend fix:** Added `_extract_player_state()` to unwrap nested `{player: {...}}`
awareness state before passing to `validate_awareness_state()`. Health endpoints
(`/health/multiplayer/players`, `/health/multiplayer/activity`) were returning
empty/default values because the validator expected flat keys. Backwards-compatible
with flat legacy format. Fixed lifecycle tests for pycrdt-websocket >= 0.16 API.

**Frontend perf:** Added `POSITION_BROADCAST_INTERVAL_MS` (60ms) throttle to position
broadcasts in `useMultiplayer`. Leading+trailing pattern: immediate broadcast when
window has elapsed, deferred trailing broadcast otherwise. Reduces bandwidth for
multi-user sessions without visible lag.

**Tests:** 8 new backend tests (extract helper, nested format endpoints, skipping
non-player states), 2 new frontend tests (throttle behaviour). All 580 frontend
and 63 backend multiplayer tests pass.

## v315 — Multiplayer Cursor Sharing

**Cursor sharing:** Remote players' mouse cursors visible in the Hand World scene
as colored SVG arrow pointers with name labels. Cursor position broadcast via Yjs
awareness `cursor` field, throttled to `CURSOR_BROADCAST_INTERVAL_MS` (100ms) with
leading+trailing pattern. `RemoteCursor` component with smooth CSS transition
interpolation (80ms). `HandWorldScene` tracks mouse via `onMouseMove`/`onMouseLeave`
and converts to scene-relative percentages.

**Tests:** 11 new frontend tests (4 useMultiplayer cursor, 4 HandWorldScene cursor,
3 RemoteCursor component). 593 frontend tests total.

## v316 — Cursor Broadcast Throttle Coverage

**Coverage:** Added 3 tests for the `updateCursor()` throttle behavior in
`useMultiplayer`, mirroring the position throttle tests from v313:
- Rapid cursor updates within `CURSOR_BROADCAST_INTERVAL_MS` are throttled with
  a trailing broadcast
- Cursor broadcasts immediately when the throttle window has elapsed
- `updateCursor(null)` (mouse leaving scene) cancels pending throttle timer and
  broadcasts immediately

**Tests:** 3 new frontend tests. 596 frontend tests total.
