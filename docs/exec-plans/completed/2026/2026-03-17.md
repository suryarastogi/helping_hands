## v247–v262 Themes

- **Provider name constants** — 20 bare provider name strings → 5 `_PROVIDER_*` constants
  in `model_provider.py` and `goose.py` (v247)
- **Git-not-found constants** — 6 bare debug strings → 3 named constants in
  `pr_description.py` (v248)
- **Env var deduplication** — `_ENV_GIT_TERMINAL_PROMPT`/`_ENV_GCM_INTERACTIVE`
  duplicated across `github_url.py` and `base.py` → canonical public exports (v249)
- **Hand factory extraction** — `_hand_from_backend()` factory function + 10 backend
  name constants in `factory.py` (v250)
- **GitHub error constants** — `_GITHUB_ERRORS` tuple + Celery `inspect` exception
  narrowing (v251)
- **Backend lookup constants** — `_BACKEND_LOOKUP` dict + factory coverage (v252)
- **Direct attribute access** — Replaced 14 defensive `getattr()` calls with direct
  attribute access across `celery_app.py`, `app.py`, `web.py`, `base.py` (v253–v254)
- **Import consolidation** — `combine-as-imports = true` in ruff isort, reducing
  24 → 3 statements in `celery_app.py` and 25 → 1 in `app.py` (v255)
- **DRY env var status** — `_env_var_status()` static method on `_TwoPhaseCLIHand`,
  replacing 5 inline `os.environ.get()` patterns (v256)
- **Centralize max iterations** — `DEFAULT_MAX_ITERATIONS` constant defined once in
  `iterative.py`, re-exported from `server/constants.py` (v257)
- **DRY env var parsing** — `_parse_positive_env_var()` helper in `pr_description.py`
  replacing duplicated parsing logic (v258)
- **DRY repo tmp + token resolution** — `repo_tmp_dir()` and `resolve_github_token()`
  shared helpers in `github_url.py`, unified `_PR_TRUTHY_VALUES` in
  `pr_description.py` (v259)
- **Unified truthy values** — Added `"on"` to base `_TRUTHY_VALUES`, removed
  `_PR_TRUTHY_VALUES` and `_CLI_TRUTHY_VALUES` duplicates, updated
  `_is_truthy_env()` to strip whitespace, added `_get_env_stripped()` helper,
  simplified 3 consumers to use `_is_truthy_env()` (v260)
- **Repo spec pattern + validation helpers** — `REPO_SPEC_PATTERN` constant
  (3 duplicates → 1), `invalid_repo_msg()` (2 duplicates → 1), `format_type_error()`
  (3 similar patterns → 1 helper) (v261)
- **Stream parser DRY** — `_OUTPUT_FORMAT_STREAM_JSON` constant (2 bare strings → 1),
  `_normalize_preview()` helper (2 duplicate `strip().replace()` → 1),
  `_extract_message_blocks()` helper (2 duplicate message extraction → 1) (v262)
