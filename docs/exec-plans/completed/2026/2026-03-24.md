## v298: Multiplayer Smooth Movement & Typing Indicator

**Smooth movement:** Added `transition: left 150ms linear, top 150ms linear` to
`.remote-player` CSS so remote avatars glide instead of snapping between
awareness updates (~6–10 Hz). Pure CSS improvement — no JS changes needed.

**Typing indicator:** Added `typing: boolean` field to Yjs awareness state.
`useMultiplayer` hook gained `setTyping()`, `isLocalTyping`, and `remoteTyping`.
`PlayerAvatar` renders a pulsing "..." bubble when typing (suppressed by
emote/chat, takes priority over idle indicator).

**10 new tests, 433 frontend tests total.**

## v299: Minimap & Chat Cooldown

**Minimap:** New `Minimap` component renders a bird's-eye overlay showing local
player (white dot), remote players (coloured dots), and active workers (amber
dots) positioned relative to scene bounds.

**Chat cooldown:** `CHAT_COOLDOWN_MS` (2 s) rate limit enforced in
`useMultiplayer.sendChat`. Chat input disabled during cooldown with "Wait..."
placeholder text. `chatOnCooldown` state threaded through `HandWorldScene` props.

**16 new tests (6 Minimap, 3 useMultiplayer cooldown, 7 HandWorldScene), 449 frontend tests total.**

## v300: Player List API & E2E Multiplayer Tests

**Player list endpoint:** New `GET /health/multiplayer/players` endpoint exposes
connected player details by reading Yjs awareness states server-side. Returns
`{players: [{player_id, name, color, x, y, idle}], count: N}`. Backend:
`get_connected_players()` and `_parse_awareness_state()` in `multiplayer_yjs.py`
with graceful fallback when pycrdt-websocket not installed.

**Playwright e2e tests:** New `frontend/e2e/multiplayer.spec.ts` with multi-context
tests verifying independent browser contexts each render Hand World with local
player avatars. Covers: two-context scene rendering, independent player name inputs,
keyboard movement position updates.

**12 new backend tests (5 _parse_awareness_state, 6 get_connected_players, 1 endpoint), 24 backend tests total.**
**3 new Playwright e2e tests.**
