## v304: Schedule PR Auto-Persist

**Auto-persist newly created PRs:** When a scheduled task with no `pr_number`
creates a new PR, the PR number is automatically saved back to the schedule so
subsequent runs push to the same PR.

**Implementation:**
- `Hand.last_pr_metadata` attribute stores finalization metadata after PR
  creation/update.
- `ScheduleManager.update_pr_number()` writes just the PR number field.
- `_maybe_persist_pr_to_schedule()` helper in `celery_app.py` bridges the
  hand result to schedule storage with guard conditions.
- `build_feature` task gains optional `schedule_id` parameter, passed from
  both `scheduled_build` and `trigger_now`.
- Non-E2E hand results now include `**hand.last_pr_metadata` for consistent
  PR metadata in task results.

**18 new tests** (4 ScheduleManager, 7 celery helper, 7 hand instantiation).

## v305: Shared World Decorations via Y.Map

**Persistent shared state:** Players can place emoji decorations in the world
scene that all connected players see in real-time. Uses Y.Map document state
(not just awareness), exercising the persistent Y.Doc capability for the first
time.

**Implementation:**
- `WorldDecoration` type, `DECORATION_EMOJIS` palette (8 emojis),
  `MAX_DECORATIONS` cap (20)
- `useMultiplayer` hook observes Y.Map `"decorations"`, exposes
  `placeDecoration(emoji, x, y)` and `clearDecorations()`
- Decoration toolbar in Factory Floor panel with emoji palette, count display,
  and clear button
- Double-click scene to place selected emoji at click position
- Pop animation and crosshair cursor during placement mode

**16 new tests** (6 hook, 10 scene).

## v306: Join/Leave Notifications & Spawn Randomization

**Multiplayer UX improvements:** System messages in chat history for player
join/leave events, and randomized spawn positions to prevent avatar overlap.

**Implementation:**
- Yjs awareness `change` event `added`/`removed` arrays drive join/leave
  system messages (`ChatMessage.isSystem` flag)
- `.chat-history-system` CSS class: italic, muted colour, hidden player name
- `randomSpawnPosition()` utility generates positions within padded bounds
- `useMovement` initializes with `useRef`-stable random spawn position
- `SPAWN_PADDING` constant (10%) prevents edge spawns
- Mock awareness in both `useMultiplayer.test.tsx` and `App.test.tsx` updated
  to pass `{added, updated, removed}` changes object

**5 new tests** (3 hook join/leave, 1 spawn bounds, 1 scene system message).
