## v273: Multiplayer Hand World

**Full-stack multiplayer:** Added `/ws/world` WebSocket endpoint in `multiplayer.py`
with `WorldConnectionManager` supporting up to 20 simultaneous players. Frontend
connects via native WebSocket API, renders remote player avatars with unique
colors, name tags, directional sprites, and walking animations. Protocol:
`players_sync`, `player_joined`, `player_left`, `player_moved`. No external
dependencies — uses FastAPI built-in WebSocket + browser WebSocket API.

**Backend tests (`test_multiplayer.py`):** 13 tests — PlayerState serialization,
connect/disconnect/broadcast, position clamping, capacity limits, color cycling,
dead connection cleanup, endpoint integration.

## v275: Multiplayer Emotes

**Emote system:** Added emote support to multiplayer Hand World. Players press
keys 1–4 to trigger emotes (wave 👋, celebrate 🎉, thumbsup 👍, sparkle ✨).
Backend validates emote names against `_VALID_EMOTES` and broadcasts
`player_emoted` messages. Frontend renders emoji bubbles that float up and
fade out over 2 seconds via CSS `@keyframes emote-float` animation.

**Backend tests:** 7 new tests — valid emote broadcast, invalid emote rejection,
unknown player noop, sender exclusion, all emotes acceptance, missing key handling,
endpoint emote integration.

**Frontend tests:** 2 new tests — local emote bubble rendering on key press,
remote emote bubble on `player_emoted` message.

## v276: Yjs-Based Multiplayer Sync

**Awareness protocol migration:** Replaced custom JSON-over-WebSocket multiplayer
protocol with Yjs awareness. Backend: `pycrdt-websocket` ASGIServer mounted at
`/ws/yjs`. Frontend: `yjs` + `y-websocket` WebsocketProvider, awareness carries
ephemeral player presence (position, direction, walking, emotes). Color/name
derived client-side from `Y.Doc.clientID`. Legacy `/ws/world` endpoint kept for
backward compatibility.

## v287: Multiplayer Chat Bubbles

**Chat system:** Added text chat to Hand World multiplayer. Players type in a chat
input (visible when connected) and press Enter to send. Chat messages appear as
speech bubbles above avatars with a float-up fade animation over 4 seconds.

**Implementation:** Chat is broadcast via the existing Yjs awareness `chat` field —
no new backend endpoints. `CHAT_DISPLAY_MS` (4s) controls bubble duration,
`CHAT_MAX_LENGTH` (120 chars) limits message length. The `useMultiplayer` hook
gained `sendChat`, `localChat`, and `remoteChats` exports.

**13 new tests:** Constants (2), PlayerAvatar (2), HandWorldScene (6),
useMultiplayer (3). **294 total frontend tests passing.**

## v290: Player Idle/AFK Detection

**Idle detection:** Players with no movement for 30 seconds (`IDLE_TIMEOUT_MS`)
are marked as idle. A floating "zzz" indicator with a gentle bob animation appears
above idle avatars (suppressed when emote or chat bubble is active). The presence
panel shows "(idle)" next to idle remote players.

**Implementation:** `useMultiplayer` tracks `lastActivityRef` (reset on any
position/direction/walking change) and runs a 5-second `setInterval` check.
`isLocalIdle` state is broadcast via Yjs awareness `idle` field. `PlayerAvatar`
renders `.idle-indicator` span when `idle && !emote && !chat`.

**13 new tests:** Constants (1), PlayerAvatar (4), HandWorldScene (3),
useMultiplayer (5). **324 total frontend tests passing.**
