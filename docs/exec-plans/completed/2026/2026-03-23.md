## v273: Multiplayer Hand World

**Full-stack multiplayer:** Added `/ws/world` WebSocket endpoint in `multiplayer.py`
with `WorldConnectionManager` supporting up to 20 simultaneous players. Frontend
connects via native WebSocket API, renders remote player avatars with unique
colors, name tags, directional sprites, and walking animations. Protocol:
`players_sync`, `player_joined`, `player_left`, `player_moved`. No external
dependencies — uses FastAPI built-in WebSocket + browser WebSocket API.

**Backend tests (`test_multiplayer.py`):** 13 tests — PlayerState serialization,
connect/disconnect/broadcast, position clamping, capacity limits, color cycling,
dead connection cleanup, endpoint integration.

## v274: Multiplayer Testing & Consolidation

**E2E test:** Added best-effort remote player rendering test to `world-view.spec.ts`.

**Consolidation:** Created 2026-03-23 daily and Week-13 weekly summaries. Marked

## v275: Multiplayer Emotes

**Emote system:** Added emote support to multiplayer Hand World. Players press
keys 1–4 to trigger emotes (wave 👋, celebrate 🎉, thumbsup 👍, sparkle ✨).
Backend validates emote names against `_VALID_EMOTES` and broadcasts
`player_emoted` messages. Frontend renders emoji bubbles that float up and
fade out over 2 seconds via CSS `@keyframes emote-float` animation.

**Backend tests:** 7 new tests — valid emote broadcast, invalid emote rejection,
unknown player noop, sender exclusion, all emotes acceptance, missing key handling,
endpoint emote integration.

**Frontend tests:** 2 new tests — local emote bubble rendering on key press,
remote emote bubble on `player_emoted` message.
